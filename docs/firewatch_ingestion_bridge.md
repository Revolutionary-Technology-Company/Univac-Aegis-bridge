Univac Aegis Central Ingestion Bridge
-------------------------------------

**Author:** Cory Andrew Hofstad

**Target:** Univac-Aegis-bridge Architecture

**Overview**

The Aegis Ingestion Bridge is a high-performance, asynchronous REST API designed to centralize life-safety telemetry from distributed Fire-Watch edge nodes. By delegating proprietary protocol decoding (e.g., Edwards RS-232 hex translation) to the local edge devices, the central Univac Aegis core remains insulated from hardware-specific serial parsing. It exclusively ingests clean, strongly-typed JSON payloads.

**Architectural Flow**

1.  **The Edge (Fire-Watch):** A local node intercepts physical panel data, decodes it, normalizes the severity, and securely transmits an `AegisTelemetryPayload`.

2.  **The Ingestion Gate (Univac Aegis):** The `aegis_ingestion_node.py` server receives the payload via an asynchronous HTTP POST request.

3.  **Validation (Pydantic):** The payload is strictly type-checked. Malformed payloads are rejected with a 422 Unprocessable Entity status before they can crash the core event loop.

4.  **Routing Matrix:** The validated payload is routed to specific asynchronous worker subroutines based on its Normalized Severity Level.

**The Telemetry Payload Model**

The bridge strictly enforces the following data contract for all incoming edge nodes:

-   **EventId** (String): Unique UUID generated by the edge node.

-   **Timestamp** (DateTime): UTC timestamp of the hardware event.

-   **SourceNode** (String): Identifier of the edge device (e.g., Edwards-Axis-AX-01).

-   **EventType** (String): Parsed event string (e.g., ALARM, TROUBLE, NORMAL).

-   **SeverityLevel** (Integer): Normalized threat level from 0 to 10.

-   **RawProtocolData** (String): Hex string of the original panel bytes for forensic auditing.

**Severity Routing Logic**

The core utilizes a flat, guard-clause-based routing matrix:

-   **Severity 8 to 10 (Critical Life-Safety):** Triggers `handle_critical_alarm`. Dispatches SIP/VoIP evacuation calls and actuates base infrastructure interlocks.

-   **Severity 4 to 7 (Hardware Trouble):** Triggers `handle_trouble_supervisory`. Logs ground faults, open circuits, and sensor failures to the maintenance engineering ledger.

-   **Severity 0 to 3 (Nominal/Reset):** Triggers `handle_normal_telemetry`. Clears active UI dashboards and resets safety latches.

**Execution Command**

To spin up the ingestion node on the central server utilizing 4 parallel asynchronous workers:

`uvicorn src.control_core.aegis_ingestion_node:app --host 0.0.0.0 --port 8000 --workers 4`

requirements.txt
================

This file defines the exact library versions required to run the Univac Aegis core processing scripts, including the new asynchronous ingestion node, the Typer CLI nodes, and the serial hardware validators.

Save this file in the root directory of your `Univac-Aegis-bridge` repository.

Plaintext

```
fastapi>=0.100.0
uvicorn>=0.23.0
pydantic>=2.0.0
typer>=0.9.0
rich>=13.4.2
pyserial>=3.5
requests>=2.31.0
numpy>=1.24.0

```

Environment Initialization
--------------------------

To install these dependencies cleanly onto your centralized deployment server without polluting the global operating system packages, execute the following commands in your engineering terminal:

`python3 -m venv aegis_env`

`source aegis_env/bin/activate`

`pip install -r requirements.txt`
